Overview

Python 3D PIC scaffold in src/. Production electrostatic runs use structure-of-arrays (SoA) particle buffers with optional Numba parallel kernels. Modules are designed for a future C++ port: explicit pushers, staggered Yee EM grids, grid-node electrostatic grids, and prescribed external E/B fields with polar-transform geometry.

Class YeeGrid

Staggered FDTD Maxwell solver with scalar and batched CIC deposit/gather (WIP driver).

Class ElectrostaticGrid

Grid-node Poisson solve, CIC deposit/gather, periodic/anode/reflecting BCs.

Class Pushers

Boris, relativistic Boris, Vay, Higuera–Cary; scalar and batch dispatch.

Module pic_kernels

Numba parallel CIC deposit, E gather, B=0 kick, and periodic wrap.

Getting Started

Run tests and serve this documentation from the repository root.

Setup: See the repository README for OpenBLAS NumPy install. Troubleshooting & manual rebuild if the build fails.
PYTHONPATH: Example drivers and unit tests prepend src/ automatically. For ad-hoc scripts, add src/ to sys.path or set PYTHONPATH=src. 
# Activate venv (see README.md for NumPy/OpenBLAS setup)
source .venv/bin/activate

# Run tests
python -m unittest discover -s tests -v

# Run an electrostatic example (no animation)
python examples/01_kr_rf_plasma.py --no-animate --steps 100

# Serve documentation
cd docs && python3 -m http.server 8000
# Open http://localhost:8000

Typical Driver Loop

One PIC timestep: deposit → solve fields → gather → push → drift → boundaries. Live electrostatic examples in examples/ use SoA arrays (pos_e, vel_e, …) and optional Numba kernels via particle_backend="numba".

# Batched electrostatic path (see examples/common.py) grid.zero_rho() grid.deposit_rho_cic_batch(pos_all, charges) # Numba if periodic + numba backend grid.solve_fields() # FFT Poisson (periodic) or sparse solve E_self = grid.gather_e_cic_batch(pos_e) # (N, 3) E_total = E_self + efield.at_batch(pos_e, t) # external field, batched when available # B=0 electrostatic kick (Numba in-place) or full Boris batch: electric_kick_b0(vel_e, E_total, q_e / m_e, dt) pos_e += vel_e * dt wrap_positions_periodic(pos_e, Lx, Ly, Lz) # Single-particle API (tests, prototyping): grid.deposit_rho_cic(x, y, z, q) E = grid.gather_e_cic(x, y, z) + efield.at(pos, t) v = Pushers.push_batch("boris", vel, E, B, q, m, dt) grid.reflect_particle(p) # reflecting / anode walls

Particle

src/Particle.py

Single macro-particle with position, velocity, charge, mass, and species kind. Velocity updates are handled externally by Pushers. Production drivers in examples/ use SoA NumPy arrays instead of Particle instances for performance.

Particle(kind, x, y, z, vx, vy, vz, q, m)

Creates a particle. pos and vel are NumPy 3-vectors stored internally.

move(dt: float) -> None

Explicit position update: pos += vel * dt.

GetterSetter
get_position()set_position(pos)
get_velocity()set_velocity(vel)
get_charge()set_charge(q)
get_mass()set_mass(m)
get_kind()set_kind(kind)

Pushers

src/Pushers.py

Explicit particle velocity integrators. All pushers accept lab-frame E, B (3-vectors), charge q, mass m, and timestep dt.

Pushers.higuera_cary(vel, E, B, q, m, dt, c=1.0) -> ndarray

Pushers.push_batch(kind, vel, E, B, q, m, dt, c=1.0) -> ndarray

Returns updated velocity array. Scalar pushers accept (3,) vectors; push_batch accepts matching (N, 3) vel, E, and B. Boris uses a vectorized kernel; other kinds fall back to row-wise scalar push until dedicated batch kernels are added.

  • lorentz_gamma_from_velocity(vel, c)
  • velocity_to_proper_velocity(vel, c)
  • proper_velocity_to_velocity(u, c)

YeeGrid

src/grids/YeeGrid.py

Staggered Yee grid for full EM PIC. Field components live on faces/edges; guard cells support periodic, anode (PEC), or reflecting (PMC-style) boundaries. Not yet wired to a live EM driver in examples/ — covered by tests/test_yee_grid.py.

YeeGrid(nx, ny, nz, dx=1, dy=1, dz=1, ng=1, boundary="periodic", eps0=1, mu0=1, anode_potential=0)

Allocates staggered Ex, Ey, Ez, Bx, By, Bz, Jx, Jy, Jz, rho with guard layers. Property c = 1/√(μ₀ε₀).

boundaryBehavior
periodicWrap fields in guard cells
anodePEC walls: tangential E = 0, normal B = 0
reflectingPMC-style: normal E = 0, tangential B = 0; particle reflection
  • step_fields(dt) — one FDTD half-step/leapfrog cycle
  • update_b(dt) / update_e(dt) — individual Maxwell updates
  • apply_boundaries() — enforce wall/periodic guards
  • zero_fields() / zero_currents()
  • cfl_dt_limit(safety=0.99) — suggested timestep from grid spacing
  • deposit_rho_cic(x, y, z, q)
  • deposit_rho_cic_batch(positions, charges) — (N, 3) + (N,); uses Numba when periodic and available
  • deposit_j_cic(x, y, z, vx, vy, vz, q)
  • gather_e_cic(x, y, z) -> ndarray
  • gather_b_cic(x, y, z) -> ndarray
  • gather_e_cic_batch(positions) -> (N, 3)
  • gather_b_cic_batch(positions) -> (N, 3)
  • position_in_domain(pos) — wrap or clamp per BC
  • reflect_particle(particle) — specular reflection at walls

ElectrostaticGrid

src/grids/ElectrostaticGrid.py

Grid-node electrostatic PIC grid: deposit ρ at vertices, solve Poisson for φ, compute E. Periodic BC uses FFT; anode (Dirichlet) and reflecting (Neumann + pinned φ) use sparse direct solve. CIC deposit/gather at integer grid nodes with offset (0, 0, 0).

ElectrostaticGrid(nx, ny, nz, dx=1, dy=1, dz=1, ng=1, boundary="periodic", eps0=1, anode_potential=0, particle_backend="numba")

Fields: rho, phi, Ex, Ey, Ez on grid nodes with guards. particle_backend is "numba" or "numpy" (Numba used for periodic CIC when available).

  • solve_poisson() — φ from ρ (FFT if periodic)
  • compute_e_field() — E from −∇φ (spectral gradient if periodic)
  • solve_fields() — both steps
  • zero_rho() / zero_fields()
  • deposit_rho_cic(x, y, z, q)
  • deposit_rho_cic_batch(positions, charges, in_place=False)
  • gather_e_cic(x, y, z) -> ndarray
  • gather_e_cic_batch(positions, in_place=False) -> (N, 3)
  • gather_fields(x, y, z, B=None) -> (E, B) — B defaults to zero
  • position_in_domain_batch(positions, in_place=False)
  • reflect_particle(particle)
  • particle_backend property — active backend after Numba availability check

ElectricFields

src/fields/ElectricFields.py

Prescribed spacetime-dependent E(r, t). Wave sources are authored in a fixed local frame (+z propagation). Spatial rotation into the PIC grid uses ElectricFields.transform only — no time-dependent frame rotation. Shares wave geometry helpers with MagneticFields via field_frame.

Polarization notes

  • delta = 0 — linear polarization (default)
  • delta = π/2, psi = π/4 — circular polarization

Legacy lab-frame APIs

at(pos, t=0.0) -> ndarray

on_grid(x, y, z, t=0.0) -> (Ex, Ey, Ez)

Combine sources with + (returns ElectricFieldsSum, which also implements at_batch). Works with PolarTransformedField as well.

WaveFrame & Polar Transform

src/fields/field_frame.py

Static local-to-lab geometry shared by electric and magnetic prescribed fields. The frame does not rotate in time; temporal behavior is in wave phase and elliptical parameters.

resolve_k_magnitude(wavevector=, k_magnitude=, wavelength=)
local_wavevector(k_magnitude) -> [0, 0, k]
evaluate_polarized_wave_local(phi, amplitude, polarization_kind, psi, delta, waveform)
normalize_envelope_width(width) -> (3,)
gaussian_envelope_local(r_local, center, width) -> float
evaluate_gaussian_pulse_local(r, t, amplitude, omega, wavevector, center, width, ...)

Used by both ElectricFields and MagneticFields for local-frame sinusoidal/elliptical waves and Gaussian pulses. The Gaussian envelope is axis-aligned in local coordinates; PolarTransformedField rotates both the envelope sampling point and the field vector into the lab frame.

WaveFrame.from_basis(k_direction, polarization=None, origin=None)
WaveFrame.from_spherical(theta, phi, pol_angle=0, origin=None)
WaveFrame.identity(origin=None)

Properties: e1, e2, k_hat (lab-frame basis vectors).

  • position_to_local(r_lab) / position_to_lab(r_local)
  • vector_to_lab(v_local) / vector_to_local(v_lab)
  • wavevector_lab(k_magnitude)
PolarTransformedField(source, frame) # alias: TransformedField

Evaluates source at local coordinates and rotates the vector into the lab frame (works for E or B):

r_local = Rᵀ (r_lab − origin)
F_lab   = R F_local(r_local, t)

Attributes: .source, .frame, .at(pos, t), .at_batch(positions, t).

Superposition via + dispatches to ElectricFieldsSum or MagneticFieldsSum based on the operand type.

Example: Electrostatic Particle Push

Single-particle API for prototyping. For full simulations use examples/01_kr_rf_plasma.py or ElectrostaticPICSimulation in examples/common.py (batched SoA + Numba). 

import sys
sys.path.insert(0, "src")

import numpy as np
from Particle import Particle
from grids import ElectrostaticGrid
from fields import ElectricFields
from Pushers import Pushers

grid = ElectrostaticGrid(32, 32, 32, dx=0.1, boundary="periodic")
efield = ElectricFields.uniform([0.0, 0.0, 1e5])

p = Particle(kind="electron", x=1.6, y=1.6, z=1.6,
               vx=0.0, vy=0.0, vz=1e5, q=-1.602e-19, m=9.109e-31)

dt = 1e-12
for step in range(100):
    t = step * dt
    grid.zero_rho()
    pos = p.get_position()
    grid.deposit_rho_cic(*pos, p.get_charge())
    grid.solve_fields()

    E = grid.gather_e_cic(*pos) + efield.at(pos, t)
    B = np.zeros(3)
    v = Pushers.push("boris", p.get_velocity(), E, B, p.get_charge(), p.get_mass(), dt)
    p.set_velocity(v)
    p.move(dt)

# Batched equivalent (N particles):
# vel = Pushers.push_batch("boris", vel, E_batch, B_batch, q, m, dt)

Example: Electromagnetic Push

Yee grid with an oblique guide field via polar transform.

import numpy as np
from grids import YeeGrid
from fields import MagneticFields, WaveFrame
from Pushers import Pushers

grid = YeeGrid(64, 64, 64, dx=1e-3, boundary="periodic")

local_B = MagneticFields.uniform([0.0, 0.0, 0.5])
frame = WaveFrame.from_spherical(theta=np.deg2rad(20), phi=0.0)
bfield = MagneticFields.transform(local_B, frame)

dt = grid.cfl_dt_limit()
# ... deposit, step_fields(dt), then per particle:
E = grid.gather_e_cic(x, y, z)
B = grid.gather_b_cic(x, y, z) + bfield.at(p.get_position(), t)
v = Pushers.push("boris_relativistic", p.get_velocity(), E, B,
                 p.get_charge(), p.get_mass(), dt)

Example: Oblique Laser via Polar Transform

Author a wave in local coordinates, rotate into the PIC cube with a static frame.

import numpy as np
from fields import ElectricFields, WaveFrame

k0 = 2 * np.pi / 800e-9
omega = 2 * np.pi * 3e14

local = ElectricFields.sinusoidal_elliptical(
    E0=1e10,
    omega=omega,
    k_magnitude=k0,
    psi=np.pi / 4,
    delta=np.pi / 2,   # circular polarization
)

frame = WaveFrame.from_spherical(
    theta=np.deg2rad(30),   # incidence polar angle from +z
    phi=np.deg2rad(45),     # azimuth in xy-plane
)

laser = ElectricFields.transform(local, frame)

pos = np.array([1e-6, 2e-6, 0.0])
E_ext = laser.at(pos, t=1e-15)

Example: Oblique Gaussian Laser Pulse

Author a finite Gaussian pulse in local coordinates (envelope along local axes, propagation along local +z), then rotate into the PIC cube with a static frame.

import numpy as np
from fields import ElectricFields, WaveFrame

k0 = 2 * np.pi / 800e-9
omega = 2 * np.pi * 3e14

local = ElectricFields.gaussian_pulse_local(
    E0=1e10,
    omega=omega,
    k_magnitude=k0,
    center=(0.0, 0.0, 0.0),
    width=(5e-6, 5e-6, 10e-6),   # w0, w0, z_R in local frame
    psi=0.0,
)

frame = WaveFrame.from_spherical(
    theta=np.deg2rad(30),
    phi=np.deg2rad(45),
)

pulse = ElectricFields.transform(local, frame)

# )

pos = np.array([1e-6, 2e-6, 0.0])

Example: Paired Oblique E + B Wave

For vacuum EM waves, B is related to E by B = (k̂ × E) / ω (SI units). Build E from the transform API, then set B amplitude and pol angle accordingly.

import numpy as np
from fields import ElectricFields, MagneticFields, WaveFrame

omega = 2 * np.pi * 3e14
k0 = omega / 3e8
E0 = 1e8
B0 = E0 / 3e8   # |B| = |E| / c for plane wave in vacuum

frame = WaveFrame.from_spherical(theta=np.deg2rad(25), phi=np.deg2rad(10))

efield = ElectricFields.transform(
    ElectricFields.sinusoidal_linear(E0=E0, omega=omega, k_magnitude=k0),
    frame,
)
bfield = MagneticFields.transform(
    MagneticFields.sinusoidal_linear(B0=B0, omega=omega, k_magnitude=k0, psi=np.pi / 2),
    frame,
)

pos = np.zeros(3)
t = 0.0
E_ext = efield.at(pos, t)
B_ext = bfield.at(pos, t)

Example: Field Superposition

Combine bias, file-backed, and transformed sources with +. Use at_batch when driving many particles.

from fields import ElectricFields, WaveFrame
import numpy as np

bias = ElectricFields.uniform([0.0, 0.0, 1e4])
file_field = ElectricFields.from_csv("data/electric_uniform_z.csv")

local_pulse = ElectricFields.sinusoidal_linear(
    E0=5e9, omega=2e15, k_magnitude=1e7,
)
frame = WaveFrame.from_spherical(theta=0.2, phi=0.0)
pulse = ElectricFields.transform(local_pulse, frame)

efield = bias + file_field + pulse
E_total = efield.at_batch(positions, t=step * dt)   # (N, 3)
